# Sistema de Internacionalização (i18n) — Documentação

**Ficheiro:** `js/i18n.js`  
**Versão:** Heródoto v7.43  
**Idiomas suportados:** Português (`pt`), Inglês (`en`), Espanhol (`es`)

---

## Visão Geral

O módulo `i18n.js` centraliza todas as strings de interface e os campos multilingues das entidades históricas. O sistema é **sem dependências externas** — não usa bibliotecas de i18n de terceiros.

---

## API Pública

```js
import { applyTranslations, setCurrentLang, getCurrentLang, getEntityLabel, getEntityField, formatarAno, TRANSLATIONS } from './i18n.js';
```

| Função / Export | Descrição |
|---|---|
| `applyTranslations(lang?)` | Aplica o idioma a todos os elementos `[data-i18n]` do DOM |
| `setCurrentLang(lang)` | Define o idioma ativo (`'pt'`, `'en'`, `'es'`) |
| `getCurrentLang()` | Retorna o idioma ativo atual |
| `getEntityLabel(entity)` | Retorna o nome da entidade no idioma atual |
| `getEntityField(entity, field)` | Retorna o campo (`descricao`, `contexto`, etc.) no idioma atual |
| `formatarAno(ano)` | Formata um número de ano para exibição (ex.: `-500` → `500 a.C.`) |
| `TRANSLATIONS` | Objeto com todas as strings dos 3 idiomas |

---

## Aplicação no HTML

Qualquer elemento com o atributo `data-i18n="chave"` tem o seu `textContent` substituído pela tradução ao chamar `applyTranslations()`:

```html
<button data-i18n="timeline_btn">Linha do Tempo</button>
<span data-i18n="filtros">Filtros</span>
```

## Estrutura de `TRANSLATIONS`

```js
export const TRANSLATIONS = {
  pt: {
    title:         'Heródoto — Visualização Histórica',
    header_title:  'Heródoto',
    filtros:       'Filtros',
    timeline_btn:  'Linha do Tempo',
    guided_btn:    'Modo Exploração Guiada',
    // ... centenas de chaves
  },
  en: { /* mesmas chaves, valores em inglês */ },
  es: { /* mesmas chaves, valores em espanhol */ },
};
```

Fallback em cascata: `TRANSLATIONS[lang][chave]` → `TRANSLATIONS['pt'][chave]` → `chave` (a própria chave como último recurso).

## Campos Multilingues nas Entidades JSON

As entidades suportam campos duplicados por idioma com sufixo `_en` / `_es`:

| Campo PT | Campo EN | Campo ES |
|---|---|---|
| `nome` | `nome_en` | `nome_es` |
| `descricao` | `descricao_en` | `descricao_es` |
| `contexto` | `contexto_en` | `contexto_es` |
| `importancia_texto` | `importancia_texto_en` | `importancia_texto_es` |

`getEntityLabel(entity)` abstrai esta lógica:
```js
function getEntityLabel(entity) {
  const lang = getCurrentLang();
  if (lang === 'en' && entity.nome_en) return entity.nome_en;
  if (lang === 'es' && entity.nome_es) return entity.nome_es;
  return entity.nome; // fallback PT
}
```

## Cobertura de Tradução

- **Strings de UI:** 100% traduzidas (PT/EN/ES)
- **Campos `nome_en` / `nome_es`:** 100% das 2.644 entidades traduzidas (concluído na sessão 14)
- **Campos `descricao_en` / `descricao_es`:** tradução parcial; em expansão

## Seletor de Idioma na UI

O selector de idioma no header chama:
```js
setCurrentLang(novoLang);
applyTranslations();
if (window.refreshTimelineLang) window.refreshTimelineLang();
if (window.updateGuidedLang)    window.updateGuidedLang();
if (window.refreshLegendLang)   window.refreshLegendLang();
```

Cada módulo que tem strings próprias expõe uma função `refresh*Lang()` registada em `window` para ser chamada após mudança de idioma.

## Adição de Novas Strings

1. Adicionar a chave nos três idiomas em `TRANSLATIONS`:
   ```js
   pt: { nova_chave: 'Texto em português' },
   en: { nova_chave: 'Text in English' },
   es: { nova_chave: 'Texto en español' },
   ```
2. Usar no HTML com `data-i18n="nova_chave"` ou no JS com `t('nova_chave')`.
